# 微信小游戏落地清单

本文用于把当前 H5 Canvas 版本迁移到微信小游戏项目。已有更细的迁移说明见 `docs/微信小游戏迁移.md`，这里按实际交付流程列 checklist。

## 1. 账号与资质

- 注册微信公众平台账号，类型选择“小程序/小游戏”。
- 获取小游戏 AppID，绑定开发者与体验成员。
- 准备软著/版权归属说明、游戏名称、图标、简介、类目、隐私政策与用户服务协议。
- 如涉及联网、登录、排行榜、支付，需要准备服务器域名并在公众平台配置合法域名。

## 2. 项目文件

最小小游戏项目需要：

```text
project.config.json
game.json
game.js
src/
assets/
```

- `game.js`：替代 H5 的 `index.html + src/main.js`，使用 `wx.createCanvas()` 创建 Canvas，再实例化 `Game`。
- `game.json`：配置方向、状态栏、网络超时等。
- `project.config.json`：微信开发者工具项目配置，包含 AppID、编译类型和基础库版本。
- `src/`：复制当前游戏源码，后续替换输入、存储、资源加载适配层。
- `assets/`：图片、音频、字体等静态资源；超过包体限制时拆分分包或 CDN。

## 3. 必改适配点

- Canvas：DOM `document.getElementById('stage')` 改为 `wx.createCanvas()`。
- 输入：键鼠事件改为触摸事件，左下虚拟摇杆 + 右下技能按钮。
- 存储：`localStorage` 改为 `wx.getStorageSync / wx.setStorageSync`。
- 网络：`fetch/WebSocket` 改为 `wx.request / wx.connectSocket`，并配置合法域名。
- 音频：Web Audio/HTMLAudio 改为 `wx.createInnerAudioContext()`。
- 生命周期：接入 `wx.onShow / wx.onHide`，暂停与恢复游戏循环。
- 适配：按 `wx.getSystemInfoSync()` 的窗口尺寸设置 Canvas，并处理刘海屏安全区。

## 4. 推荐目录

```text
wechat-game/
├── game.js
├── game.json
├── project.config.json
├── src/
│   ├── core/
│   ├── data/
│   ├── entities/
│   ├── scenes/
│   ├── systems/
│   └── ui/
└── assets/
    ├── images/
    └── audio/
```

## 5. 测试与发布流程

1. 微信开发者工具创建小游戏项目，填入 AppID。
2. 复制 `src/`，新增 `game.js/game.json/project.config.json`。
3. 完成 Canvas、输入、存储、音频适配。
4. 在开发者工具中进行模拟器测试、真机预览、体验版测试。
5. 检查首包体积、帧率、内存、弱网、切后台恢复。
6. 上传代码，填写版本号和更新说明。
7. 在公众平台提交审核，补齐隐私政策、用户协议、未成年人防沉迷与类目资料。
8. 审核通过后灰度发布，再全量发布。

## 6. 当前项目迁移建议

- 先保持 Canvas2D 与原生 JS，不急着引入引擎，迁移成本最低。
- 把 `Input` 抽成 `KeyboardMouseInput` 与 `TouchInput` 两种实现。
- 把 `Storage`、`Audio`、`AssetLoader` 做成平台适配层，H5 与微信各一套实现。
- 地图、英雄、敌人、道具已经在 `src/data/`，适合继续扩展为 JSON/远端配置。
- 当前 `MapRuntime` 已将地形渲染、碰撞和区域效果集中管理，后续新增陷阱、机关、可破坏物时优先扩展这里。
