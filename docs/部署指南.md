# 部署指南

本指南覆盖 **GitHub Pages**、**Vercel** 与 **微信小游戏** 三种发布渠道。

## 一、GitHub Pages（推荐，免费 + 自动 CI）

### 1. 准备

```bash
# 1) 在 GitHub 创建仓库 escape-from-lebethel（公开）
# 2) 把本项目推上去
cd game
git init
git add .
git commit -m "feat: initial release"
git branch -M main
git remote add origin https://github.com/<your-name>/escape-from-lebethel.git
git push -u origin main
```

### 2. 启用 Pages

仓库 → Settings → Pages：
- **Source** 选择 `GitHub Actions`
- 保存

### 3. CI 自动部署

本仓库已包含 `.github/workflows/deploy.yml`，每次 `push` 到 `main` 自动构建并发布。

发布地址形如：

```
https://<your-name>.github.io/escape-from-lebethel/
```

### 4. 自定义域名（可选）

仓库 Settings → Pages → Custom domain，填入 `game.example.com`，并在 DNS 加 `CNAME` 解析到 `<your-name>.github.io`。

---

## 二、Vercel（免费 + CDN 全球加速）

```bash
# 安装 Vercel CLI
npm i -g vercel

cd game
vercel       # 登录并跟随提示，第一次部署
vercel --prod  # 推到生产
```

或用本仓库的 `/vercel-deploy` skill 一键部署。

---

## 三、微信小游戏

详见 [微信小游戏迁移.md](微信小游戏迁移.md)，要点：

1. 注册微信公众平台 → 选择"小游戏"类目（个人/企业主体均可）
2. 下载 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
3. 把 `src/` 与 `index.html` 改造为 `game.js` + `game.json`
4. 提交审核（一般 1~3 个工作日）

---

## 四、本地预览

```bash
cd game
# 任选其一
npx http-server -p 8080 -c-1
python -m http.server 8080
```

浏览器访问 <http://localhost:8080>。

---

## 五、生产环境优化（可选）

如果要进一步提升加载速度，可使用 Vite 构建：

```bash
npm i -D vite terser
# 增加 vite.config.js 后
npx vite build  # 输出到 dist/，再发布 dist/
```

构建后体积一般可压缩到 50KB 以内（gzip）。
