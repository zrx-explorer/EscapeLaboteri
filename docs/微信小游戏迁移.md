# 微信小游戏迁移指南

> 把本 H5 项目迁移到微信小游戏的实操步骤。

## 一、概念差异

| 对比项 | H5 浏览器 | 微信小游戏 |
|---|---|---|
| 入口 | `index.html` | `game.js` |
| 配置 | 无 | `game.json` |
| Canvas | DOM 元素 | `wx.createCanvas()` |
| 网络 | `fetch` | `wx.request` |
| 存储 | `localStorage` | `wx.getStorageSync` |
| 资源 | 相对路径 | 子包 / CDN |

## 二、准备

1. 注册：[微信公众平台](https://mp.weixin.qq.com/) → 选择"小游戏"类目。
2. 下载工具：[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
3. 获取 AppID（小游戏专属）。

## 三、最小改造步骤

### 1) 新建小游戏项目

打开开发者工具 → 选择 AppID → 模板选 "JS - 经典小游戏"，得到：

```
game.js
game.json
project.config.json
```

### 2) 把本项目代码搬过去

```
小游戏根目录/
├── game.js                  # 入口（替换为下面的代码）
├── game.json
├── project.config.json
└── src/                     # 直接拷贝过来
    ├── core/  ...
    ├── data/  ...
    └── ...
```

### 3) `game.js` 入口

```js
// game.js  —— 微信小游戏入口
import { Game } from './src/core/Game.js';

const canvas = wx.createCanvas();        // 全局 Canvas
const ctx    = canvas.getContext('2d');

const sysInfo = wx.getSystemInfoSync();
canvas.width  = sysInfo.windowWidth;
canvas.height = sysInfo.windowHeight;

const game = new Game(canvas, ctx);
game.start();

// 触摸事件 → 转化为虚拟摇杆 + 技能轮盘
wx.onTouchStart((e)=> game.input.onTouchStart(e));
wx.onTouchMove ((e)=> game.input.onTouchMove(e));
wx.onTouchEnd  ((e)=> game.input.onTouchEnd(e));
```

### 4) 输入层适配

把 `src/core/Input.js` 中的 `keydown / mousemove` 替换为基于触摸事件的虚拟摇杆与按钮。

### 5) 存储层

把 `localStorage.setItem` 替换为 `wx.setStorageSync`。

> 项目中已抽离 `Storage` 接口，仅需替换实现：

```js
// src/utils/Storage.js
export const Storage = {
  get(k){ return wx.getStorageSync(k) },
  set(k, v){ wx.setStorageSync(k, v) }
}
```

### 6) `game.json`（最小）

```json
{
  "deviceOrientation": "portrait",
  "showStatusBar": false,
  "networkTimeout": { "request": 5000 }
}
```

## 四、发布上线

1. 开发者工具 → 上传代码（带版本号 + 备注）。
2. 公众平台 → 版本管理 → 提交审核。
3. 配置 ICP / 隐私政策 / 防沉迷。
4. 审核通过后发布上线（个人主体仅支持非营利类目）。

## 五、注意事项

- 单包大小 ≤ 4 MB；超出需用 **分包加载** 或 CDN。
- 不允许动态执行远程脚本（去掉 `eval` 与远程 `import`）。
- 不允许出现"分享得抽奖"等诱导行为。
- 必须接入微信登录 + 用户服务条款 + 隐私政策。

## 六、性能建议

- 图片资源压缩为 webp。
- 长期不动的元素离屏 Canvas 缓存。
- 帧率锁 30fps（小游戏开放 60fps，但中低端机表现更稳）。
- 避免使用 ES2020+ 新语法，建议构建时降级到 ES6（Babel）。
